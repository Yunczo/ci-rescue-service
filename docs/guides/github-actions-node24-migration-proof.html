Clearly labeled synthetic before and after.

Synthetic BEFORE · temporary Node 20 path

name: CI
on: [push, pull_request]

permissions:
  contents: read

jobs:
  test:
    runs-on: ubuntu-latest
    env:
      ACTIONS_ALLOW_USE_UNSECURE_NODE_VERSION: true
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version-file: .nvmrc
          cache: npm
      - run: npm ci
      - run: npm test

Limits and non-generalization.

  • This is a synthetic example, not a paid-client result, endorsement, benchmark, or success guarantee.
  • It covers two GitHub-owned JavaScript actions on a GitHub.com-hosted ubuntu-latest job. It does not cover third-party, local, Docker, or composite actions.
  • It does not test pinned commit SHAs, self-hosted runner versions, macOS or ARM32 constraints, GitHub Enterprise Server, reusable workflows, artifacts, services, matrices, secrets, permissions beyond contents: read, or external dependencies.
  • Major action upgrades can change inputs, defaults, prerequisites, and behavior. Read every relevant release note and use immutable reviewed SHAs when policy requires them.
  • A clean static review and a green repository-local run are prerequisites and evidence, not proof of future hosted behavior. Run the exact job path and inspect its first actionable failure.
  • This is migration guidance, not a security review. Never submit credentials, private logs, proprietary source, or production details.
